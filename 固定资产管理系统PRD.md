## 1. 产品概述

固定资产管理系统用于企业内部固定资产的台账维护、领用派发、借用归还、退库和领用人变更管理。系统以后台管理 Web 形态提供，重点解决资产信息分散、状态不清、流转记录缺失、人员权限混乱等问题。

### 1.1 建设目标

- 建立统一的固定资产台账，记录资产编码、名称、分类、品牌、型号、存放地点、状态、领用人等核心信息。
- 支持资产从闲置到派发、借用、归还、退库、领用人变更的基础流转。
- 支持用户、角色、权限和基础参数维护，保证不同岗位只能访问授权功能。
- 提供清晰、稳定、可直接开发的功能范围，暂不引入复杂财务和审批能力。

### 1.2 用户对象

| 用户角色 | 主要职责 | 可用功能 |
| --- | --- | --- |
| 系统管理员 | 系统初始化、人员权限配置、全局数据维护 | 登录、用户管理、角色管理、权限管理、参数设置、全部资产功能 |
| 资产管理员 | 固定资产日常维护和流转处理 | 资产台账、资产新增/修改、派发、借用、归还、退库、领用人变更 |
| 普通用户/领用人 | 作为资产领用、借用对象 | 本期默认不提供复杂自助端，可作为资产流转中的人员数据 |

### 1.3 本期范围

本期包含：登录、首页、用户管理、角色管理、权限管理、资产参数设置、固定资产台账、资产派发、资产借用、借用归还、领用退库、变更领用人。

本期不包含：复杂审批流、财务折旧、财务凭证、采购入库、报废审批、二维码盘点、移动端、消息通知、组织架构同步。

## 2. 业务规则

### 2.1 资产状态

| 状态 | 说明 | 允许操作 |
| --- | --- | --- |
| 闲置 | 资产未被领用或借用，可由资产管理员分配 | 修改、派发、借用 |
| 已派发 | 资产已分配给领用人长期使用 | 修改、退库、变更领用人 |
| 借用中 | 资产被临时借用，需按计划归还 | 修改、归还 |
| 已退库 | 资产从领用人处退回库房 | 修改、派发、借用 |

### 2.2 编码与唯一性

- 用户账号不可重复。
- 角色名称不可重复。
- 资产编码不可重复，建议由后端生成，也允许管理员手动录入后校验唯一。
- 参数名称在同一参数类型下不可重复。

### 2.3 基础校验

- 所有新增/编辑弹窗均需校验必填项。
- 删除数据前需二次确认。
- 被资产引用的基础参数不允许删除。
- 非法状态下的流转操作不允许提交，并返回明确提示。

## 3. 功能需求

### 3.1 登录

| 项目 | 需求 |
| --- | --- |
| 入口 | `index.html` 登录页 |
| 字段 | 账号、密码 |
| 操作 | 输入账号密码，点击“登录系统”进入首页 |
| 校验 | 账号、密码必填；错误时提示“账号或密码错误” |
| 成功结果 | 进入后台首页，显示当前登录用户和可访问菜单 |

### 3.2 首页/工作台

- 展示资产总数、闲置资产、已派发资产、借用中资产等统计。
- 展示待处理事项：待归还、近期新增、状态异常等静态提醒。
- 提供常用入口：新增资产、资产派发、借用登记、归还登记。

### 3.3 用户管理

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 用户账号 | 登录账号，唯一 | 是 |
| 用户姓名 | 用户真实姓名 | 是 |
| 手机号 | 联系方式 | 否 |
| 所属部门 | 用户所在部门 | 否 |
| 角色 | 绑定系统角色 | 是 |
| 状态 | 启用/停用 | 是 |

功能要求：
- 支持按账号、姓名、角色、状态查询。
- 支持新增、修改、删除、启用、停用。
- 删除前需确认；停用用户不可登录。

### 3.4 角色管理

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 角色名称 | 角色显示名称，唯一 | 是 |
| 角色编码 | 系统内部编码 | 是 |
| 角色描述 | 角色用途说明 | 否 |
| 状态 | 启用/停用 | 是 |

功能要求：
- 支持新增、修改、删除角色。
- 支持为角色分配菜单权限。
- 已绑定用户的角色删除时需提示失败或要求先解除绑定。

### 3.5 权限管理

- 以菜单树方式展示权限节点。
- 支持勾选权限并保存到角色。
- 权限至少覆盖：首页、人员管理、基础参数设置、固定资产管理、资产流转。
- 前端根据用户权限展示菜单；后端接口仍需进行权限校验。

### 3.6 资产参数设置

| 参数类型 | 示例 |
| --- | --- |
| 资产分类 | 电脑设备、办公家具、网络设备、办公电器 |
| 品牌 | 联想、戴尔、惠普、苹果 |
| 计量单位 | 台、个、套、张 |
| 存放地点 | 总部库房、财务部、研发部、会议室 |
| 使用状态 | 闲置、已派发、借用中、已退库 |

功能要求：
- 支持按参数类型查询。
- 支持参数新增、修改、删除。
- 参数被资产引用时不允许删除。

### 3.7 固定资产台账

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 资产唯一编号 | 是 |
| 资产名称 | 资产名称 | 是 |
| 资产分类 | 来自参数配置 | 是 |
| 品牌 | 来自参数配置 | 否 |
| 型号 | 资产规格型号 | 否 |
| 序列号 | 厂商序列号 | 否 |
| 计量单位 | 来自参数配置 | 是 |
| 购置日期 | 资产购入日期 | 否 |
| 存放地点 | 来自参数配置 | 是 |
| 当前状态 | 闲置/已派发/借用中/已退库 | 是 |
| 领用人 | 已派发时记录 | 否 |
| 借用人 | 借用中时记录 | 否 |
| 备注 | 补充说明 | 否 |

功能要求：
- 支持按资产编码、资产名称、分类、状态、领用人查询。
- 支持新增、修改、查看详情。
- 闲置和已退库资产可派发或借用。
- 已派发资产可退库或变更领用人。
- 借用中资产可归还。

### 3.8 资产派发

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 选择闲置或已退库资产 | 是 |
| 领用人 | 资产长期使用人员 | 是 |
| 领用部门 | 领用人所属部门 | 否 |
| 派发日期 | 默认当天 | 是 |
| 存放地点 | 派发后的使用地点 | 是 |
| 备注 | 派发说明 | 否 |

提交成功后，资产状态更新为“已派发”，记录领用人、领用部门、派发日期和使用地点。

### 3.9 资产借用

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 选择闲置或已退库资产 | 是 |
| 借用人 | 临时借用人员 | 是 |
| 借用部门 | 借用人所属部门 | 否 |
| 借用日期 | 默认当天 | 是 |
| 预计归还日期 | 借用计划归还日期 | 是 |
| 借用原因 | 借用说明 | 否 |

提交成功后，资产状态更新为“借用中”，记录借用人和预计归还日期。

### 3.10 借用归还

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 选择借用中资产 | 是 |
| 归还日期 | 默认当天 | 是 |
| 归还地点 | 资产归还地点 | 是 |
| 资产状况 | 完好/损坏/需维修 | 是 |
| 备注 | 归还说明 | 否 |

提交成功后，资产状态更新为“已退库”，清空借用人信息，记录归还信息。

### 3.11 领用退库

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 选择已派发资产 | 是 |
| 原领用人 | 系统自动带出 | 是 |
| 退库日期 | 默认当天 | 是 |
| 退库地点 | 退回库房或指定地点 | 是 |
| 资产状况 | 完好/损坏/需维修 | 是 |
| 备注 | 退库说明 | 否 |

提交成功后，资产状态更新为“已退库”，清空领用人信息。

### 3.12 变更领用人

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 资产编码 | 选择已派发资产 | 是 |
| 原领用人 | 系统自动带出 | 是 |
| 新领用人 | 新的长期使用人员 | 是 |
| 变更日期 | 默认当天 | 是 |
| 使用地点 | 新领用地点 | 是 |
| 变更原因 | 变更说明 | 否 |

提交成功后，资产状态保持“已派发”，领用人更新为新领用人，并记录变更历史。

## 4. 页面需求

静态页面采用后台管理布局：登录页、左侧菜单、顶部栏、查询区、数据表格、弹窗表单、操作按钮。页面包为纯 HTML/CSS/JS，无需构建工具，可直接本地打开。

| 页面 | 文件 | 说明 |
| --- | --- | --- |
| 登录页 | `index.html` | 账号密码登录入口 |
| 首页 | `dashboard.html` | 资产统计和快捷入口 |
| 用户管理 | `users.html` | 用户增删改查 |
| 角色管理 | `roles.html` | 角色维护与权限分配入口 |
| 权限管理 | `permissions.html` | 菜单权限树 |
| 参数设置 | `asset-params.html` | 资产参数维护 |
| 固定资产台账 | `assets.html` | 资产列表、新增、编辑、详情 |
| 资产派发 | `asset-distribute.html` | 闲置资产派发登记 |
| 资产借用 | `asset-borrow.html` | 资产临时借用登记 |
| 借用归还 | `asset-return.html` | 借用资产归还登记 |
| 领用退库 | `asset-revert.html` | 已派发资产退库登记 |
| 变更领用人 | `asset-owner-change.html` | 已派发资产领用人变更 |

## 5. 建议接口清单

本期静态页面不包含真实接口调用，后端开发可参考以下接口划分：

| 模块 | 接口方向 |
| --- | --- |
| 登录 | `POST /api/auth/login`、`POST /api/auth/logout` |
| 用户 | `GET/POST/PUT/DELETE /api/users` |
| 角色 | `GET/POST/PUT/DELETE /api/roles`、`PUT /api/roles/{id}/permissions` |
| 权限 | `GET /api/permissions/tree` |
| 参数 | `GET/POST/PUT/DELETE /api/asset-params` |
| 资产 | `GET/POST/PUT /api/assets`、`GET /api/assets/{id}` |
| 流转 | `POST /api/assets/{id}/distribute`、`POST /api/assets/{id}/borrow`、`POST /api/assets/{id}/return`、`POST /api/assets/{id}/revert`、`POST /api/assets/{id}/owner-change` |

## 6. 非功能需求

- 权限：前端控制菜单显示，后端必须校验接口权限。
- 日志：新增、修改、删除、流转操作需记录操作人、操作时间、操作内容。
- 性能：列表查询默认分页，每页 10/20/50 条。
- 兼容：支持主流桌面浏览器，推荐 Chrome、Edge。
- 数据安全：密码加密存储，敏感操作需登录态校验。
- 可维护：参数项可配置，避免在代码中写死业务枚举。

## 7. 验收标准

| 模块 | 验收场景 |
| --- | --- |
| 登录 | 正确账号可进入首页，错误账号或密码提示失败 |
| 用户管理 | 可新增、修改、删除、启用、停用用户，账号不可重复 |
| 角色权限 | 可维护角色并分配菜单权限，不同角色看到的菜单符合授权 |
| 参数设置 | 可维护参数，被引用参数删除时提示失败 |
| 资产台账 | 可新增和编辑资产，资产编码唯一，列表查询正常 |
| 资产派发 | 闲置资产派发后状态变为已派发并记录领用人 |
| 资产借用 | 闲置资产借用后状态变为借用中并记录预计归还日期 |
| 借用归还 | 借用中资产归还后状态变为已退库 |
| 领用退库 | 已派发资产退库后状态变为已退库 |
| 领用人变更 | 已派发资产变更后领用人更新，状态仍为已派发 |
| 静态页面 | 所有页面可本地打开，菜单跳转、弹窗、提示和表格展示正常 |
