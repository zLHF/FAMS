## 1. 产品概述

固定资产管理系统用于企业内部固定资产的台账维护、领用派发、借用归还、退库和领用人变更管理。系统以后台管理 Web 形态提供，支持多租户隔离、三层 RBAC 权限体系、算力网络分布式管理平台 SSO 单点登录集成。

### 1.1 建设目标

- 建立统一的固定资产台账，记录资产编码、名称、分类、品牌、型号、存放地点、状态、领用人等核心信息。
- 支持资产从闲置到派发、借用、归还、退库、领用人变更的基础流转。
- 支持三层权限体系（系统管理员、租户管理员、资产管理员），保证不同角色只能访问授权功能。
- 支持多租户数据隔离，每个租户独立的用户、角色、资产数据。
- 支持算力网络分布式管理平台 SSO 单点登录，自动开通租户和用户。
- 提供清晰、稳定、可直接开发的功能范围，暂不引入复杂财务和审批能力。

### 1.2 用户角色

| 用户角色 | 角色编码 | 主要职责 | 可用功能 |
| --- | --- | --- | --- |
| 系统管理员 | `system_admin` | 全局管理，可切换到所有租户 | 登录、所有租户管理、用户管理、角色管理、权限管理、参数设置、全部资产功能 |
| 租户管理员 | `tenant_admin` | 管理本租户用户和配置 | 登录、本租户用户管理、角色管理、权限管理、参数设置、全部资产功能 |
| 资产管理员 | `asset_manager` | 固定资产日常维护和流转处理 | 登录、资产台账、资产新增/修改、派发、借用、归还、退库、领用人变更 |

**系统管理员特性**：
- 不属于任何特定租户（全局角色，`tenant_id=NULL`）
- 登录后右上角显示所有租户的下拉切换框
- 可自由切换到任意租户进行管理，切换时自动获得该租户的管理权限
- 可新建租户，新建后立即出现在切换列表中

**租户管理员特性**：
- 只能看到和管理自己所属租户的数据
- 可在本租户内进行用户管理和权限分配

### 1.3 本期范围

本期包含：登录（含 SSO）、首页工作台、用户管理、角色管理、权限管理、租户管理、资产参数设置、固定资产台账、资产派发、资产借用、借用归还、领用退库、变更领用人、操作日志。

本期不包含：复杂审批流、财务折旧、财务凭证、采购入库、报废审批、二维码盘点、原生移动端、消息通知、组织架构同步。

## 2. 业务规则

### 2.1 多租户规则

- 每个租户拥有独立的用户、角色、资产、参数数据，租户间完全隔离。
- 系统管理员可跨租户管理，普通用户只能访问所属租户数据。
- 租户编码全局唯一，新建租户时自动创建「租户管理员」和「资产管理员」两个默认角色。
- SSO 开通时，相同企业名称匹配到同一租户，相同平台账号匹配到同一用户。
- 租户下有活跃成员时不允许删除该租户。

### 2.2 资产状态

| 状态 | 说明 | 允许操作 |
| --- | --- | --- |
| 闲置 | 资产未被领用或借用，可由资产管理员分配 | 修改、派发、借用 |
| 已派发 | 资产已分配给领用人长期使用 | 修改、退库、变更领用人 |
| 借用中 | 资产被临时借用，需按计划归还 | 修改、归还 |
| 已退库 | 资产从领用人处退回库房 | 修改、派发、借用 |

### 2.3 编码与唯一性

- 用户账号全局不可重复。
- 角色名称在同一租户内不可重复。
- 资产编码在同一租户内不可重复。
- 参数名称在同一参数类型、同一租户下不可重复。
- 租户编码全局不可重复。

### 2.4 SSO 登录规则

- 用户通过算力网络平台点击 FAMS 应用时，URL 携带 `code` 参数自动登录。
- SSO 登录时若本地无对应用户，自动创建用户并分配默认租户。
- SSO 创建的用户用户名加 `plt_` 前缀，标记为 SSO 用户，无需本地密码。
- SSO 功能通过 `SSO_ENABLED` 环境变量控制开关。

### 2.5 基础校验

- 所有新增/编辑弹窗均需校验必填项。
- 删除数据前需二次确认。
- 被资产引用的基础参数不允许删除。
- 非法状态下的流转操作不允许提交，并返回明确提示。

## 3. 功能需求

### 3.1 登录

| 项目 | 需求 |
| --- | --- |
| 入口 | 登录页 `/login` |
| 字段 | 账号、密码 |
| 操作 | 输入账号密码，点击"登录系统"进入首页 |
| 校验 | 账号、密码必填；错误时提示"账号或密码错误"；连续失败超过阈值临时锁定 |
| 成功结果 | 进入后台首页，系统管理员显示租户切换下拉框，普通用户直接进入所属租户 |

**SSO 登录流程**：

1. 用户在算力网络平台点击 FAMS 应用
2. 平台生成 `code` 参数拼接到 FAMS URL：`http://fams-url/?code=MjAyNDAxMDQxMDI4MTkwODI=`
3. FAMS 前端检测到 `code` 参数，跳转 SSO 回调页面
4. 后端调用平台身份校验接口（`appAuthCheck`）获取平台 token
5. 后端调用平台用户信息接口（`getUserInfoByToken`）获取用户信息
6. 后端匹配/创建本地用户，生成 JWT token 完成登录

| 接口 | 方向 | 路由 | 说明 |
| --- | --- | --- | --- |

## 6. 非功能需求

- **权限**：前端控制菜单显示，后端必须校验接口权限。系统管理员始终通过所有 `role_required` 检查。
- **多租户隔离**：所有业务数据按 `tenant_id` 隔离，系统管理员无租户时 API 安全降级返回空数据。
- **SSO 安全**：Open API 接口通过 `appKey + appSecret` 双重验证；心跳接口使用 HMAC-SHA256 签名认证。
- **日志**：新增、修改、删除、流转操作需记录操作人、操作时间、操作内容。
- **性能**：列表查询默认分页，每页 10/20/50 条，上限 100 条。
- **兼容**：支持主流桌面浏览器，推荐 Chrome、Edge。响应式布局支持移动端。
- **数据安全**：密码 Bcrypt 加密存储，JWT Token 含黑名单机制，登录失败锁定。
- **可维护**：参数项可配置，避免在代码中写死业务枚举。
- **心跳保活**：应用每 5 分钟向平台上报心跳，15 分钟未上报标记为离线。

- Python 3.10+
- Node.js 20.19+（构建前端）
- SQLite（开发）/ PostgreSQL / MySQL（生产推荐）

### 7.2 环境变量配置

| 变量 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `FLASK_ENV` | 否 | `development` / `production` | `development` |
| `SECRET_KEY` | 生产必填 | Flask 会话密钥 | 开发用默认值 |
| `JWT_SECRET` | 生产必填 | JWT 签名密钥，≥32 字符 | 开发用默认值 |
| `DATABASE_URL` | 否 | 数据库连接串 | SQLite |
| `CORS_ORIGINS` | 否 | 允许的前端源，逗号分隔 | `http://localhost:5173` |
| `PLATFORM_APP_ID` | SSO 必填 | 算力网络平台应用 ID | 空 |
| `PLATFORM_APP_SECRET` | SSO 必填 | 算力网络平台应用密钥 | 空 |
| `PLATFORM_BASE_URL` | SSO 必填 | 平台代理地址 | 空 |
| `PLATFORM_APP_CODE` | SSO 必填 | 应用编码 | 空 |
| `SSO_ENABLED` | 否 | SSO 功能开关 | `false` |
| `HEARTBEAT_ENABLED` | 否 | 心跳上报开关 | `false` |

### 7.3 部署方式

- **推荐**：Gunicorn + Nginx 反向代理，前端静态文件由 Nginx 提供。
- 前端构建产物在 `frontend/dist/`，后端运行在 `127.0.0.1:5001`。
- Nginx 需同时代理 `/api/` 和 `/open-api/` 路径到后端。
- 平台使用 HTTP 协议，应用地址需使用 HTTP（非 HTTPS）以避免 iframe 限制。

## 8. 验收标准

| 模块 | 验收场景 |
| --- | --- |
| 登录 | 正确账号可进入首页，错误账号提示失败，连续失败锁定 |
| SSO 登录 | 平台携带 code 参数访问时自动登录，首次访问自动创建用户 |
| 系统管理员 | 登录后右上角显示所有租户切换框，可切换到任意租户管理 |
| 租户管理员 | 只能看到和管理自己租户的用户和数据 |
| 租户管理 | 可新增租户，新增后切换下拉框立即显示；可查看租户成员 |
| 用户管理 | 可新增、修改、删除、启用、停用用户，账号不可重复 |
| 角色权限 | 可维护角色并分配菜单权限，不同角色看到的菜单符合授权 |
| 参数设置 | 可维护参数，被引用参数删除时提示失败 |
| 资产台账 | 可新增和编辑资产，资产编码唯一，列表查询正常，按租户隔离 |
| 资产派发 | 闲置资产派发后状态变为已派发并记录领用人 |
| 资产借用 | 闲置资产借用后状态变为借用中并记录预计归还日期 |
| 借用归还 | 借用中资产归还后状态变为已退库 |
| 领用退库 | 已派发资产退库后状态变为已退库 |
| 领用人变更 | 已派发资产变更后领用人更新，状态仍为已派发 |
| 操作日志 | 关键操作均有日志记录，按租户隔离 |
| 租户切换 | 系统管理员切换租户后所有数据刷新为新租户数据 |
